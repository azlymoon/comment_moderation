Сервис модерации комментариев
================================

Добро пожаловать в документацию проекта модерации комментариев. Здесь представлены
руководства для пользователей и разработчиков, а также подробная инструкция по установке
и развёртыванию системы.

Краткое описание
----------------

Сервис предназначен для автоматизированной проверки пользовательских комментариев
на токсичность и другие нарушения. Архитектура построена на FastAPI и включает:

* публичный API для интеграции с внешними веб-сервисами;
* административный интерфейс для операторов (управление заявками, категориями, правилами,
  API-ключами и административными пользователями);
* модуль машинного обучения на базе модели ``unitary/toxic-bert`` и дополнительных
  эвристик (ключевые слова, анализ тональности);
* подсистему статистики для оценки эффективности модерации.

Основные возможности
--------------------

* Приём и асинхронная обработка текстовых комментариев.
* Управление веб-сервисами интеграции и выписка API-ключей.
* Настройка категорий нарушений и правил автоматического реагирования.
* Ведение истории заявок, ручное переопределение решений, хранение версий модели.
* Сбор агрегированной статистики по каждому сервису.

Технологический стек
--------------------

* **FastAPI** – веб-фреймворк.
* **SQLAlchemy 2.0** + **PostgreSQL** (с fallback на SQLite) – хранение данных.
* **Pydantic** – валидация и сериализация моделей API.
* **Transformers** / **torch** – ML-модель токсичности.
* **Passlib** / **bcrypt** – безопасное хранение паролей и ключей.
